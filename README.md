# 智能筛选选号系统阅读文档

这是一个本地运行的福彩 3D 智能筛选/选号工具。系统重点是把常用筛选条件做成可视化按钮和分组面板，帮助用户从号码池中快速过滤出符合条件的号码，并支持结果复制、导出、历史保存和开奖回查。

> 当前版本优先保证前端交互清晰，不重写底层筛选算法。所有筛选条件之间默认是 **同时满足** 的关系。

## 1. 快速运行

在项目目录运行：

```powershell
python app.py
```

默认访问地址：

```text
http://127.0.0.1:5000/
```

常用验证命令：

```powershell
python -m pytest
```

## 2. 页面结构

页面采用“顶部状态栏 + 左侧筛选区 + 右侧结果区”的中后台工具台结构。

- 顶部状态栏：显示系统名称、最新期号、下一期号、开奖号码、本地开奖数据数量。
- 左侧筛选区：按功能分为基础输入、基础形态、统计特征、区间组合等折叠面板。
- 右侧结果区：显示号码池总数、筛选命中数、命中占比、已启用条件数量、号码墙、筛选步骤、历史记录。
- 小屏适配：窄屏下筛选区会变成可收起面板，主要按钮保留触控尺寸。

## 3. 推荐使用流程

1. 选择号码池模式，或粘贴手动号码池。
2. 设置胆码、杀码、定位、形态、统计、段组、N 码等条件。
3. 查看左侧底部的“已启用条件摘要”。
4. 点击“开始筛选”。
5. 在右侧复制号码、调整每行数量/分隔符，或保存历史。
6. 开奖数据更新后，可对历史记录做中奖回查。

## 4. 号码池模式

基础输入区支持以下号码池模式：

| 模式 | 含义 |
| --- | --- |
| 直选全量 | 生成 `000-999` 的全部直选号码 |
| 组选大底 | 生成非豹子组选代表号 |
| 组三 | 只生成两位相同的组选代表号 |
| 组六 | 只生成三位不同的组选代表号 |
| 豹子 | 只生成三位相同号码 |
| 手动号码池 | 从文本中自动提取三位数字号码 |

注意：定位筛选是百位、十位、个位的直选位置逻辑，建议配合“直选全量”或“手动号码池”使用。组选模式下位置含义不明确，系统会限制这类组合。

## 5. 基础输入

### 手动号码池

可以粘贴任意文本，系统会自动提取其中的三位数字，例如 `012`、`357`、`888`。重复号码会去重。

辅助按钮：

- 粘贴号码
- 复制号码
- 排序去重
- 填充当前大底

### 复式码生成

现在不再使用固定码数下拉框。用户直接选择数字，选中数量就是复式码长度。

- 支持选择 3 到 8 个不同数字。
- 点击“添加已选号码”后加入复式码列表。
- 点击“生成并填入号码池”后，系统生成对应的组六和组三组合，并写入手动号码池。

### 胆码筛选

胆码表示号码中至少命中一个所选数字。

例：胆码 `357`，号码中包含 `3`、`5`、`7` 任意一个即可保留。

### 杀码筛选

杀码表示号码中不能出现所选数字。

例：杀码 `48`，号码中只要出现 `4` 或 `8` 就会被过滤。

### 定位筛选

支持百位、十位、个位分别设置：

- 保留数字：该位置必须是所选数字之一。
- 排除数字：该位置不能是所选数字。

同一位置不能同时保留和排除同一个数字。

## 6. 基础形态

### 形态模式

形态模式支持“包含 / 排除”两种方式。

可选形态包括：

- 豹子
- 组三
- 组六
- 对子
- 连号
- 半连号
- 质合

质数数量单独支持 `0 / 1 / 2 / 3`，其中 `2、3、5、7` 算质数，`0、1、4、6、8、9` 算合数。

### 比例形态：直选读法

比例形态是按百位、十位、个位顺序读取，属于直选格式。

| 条件 | 示例 | 含义 |
| --- | --- | --- |
| 大小 | 大-小-大 | 百位大、十位小、个位大 |
| 奇偶 | 奇-偶-奇 | 百位奇、十位偶、个位奇 |
| 质合 | 质-合-质 | 百位质、十位合、个位质 |
| 012 路 | 0-1-2 | 百位 0 路、十位 1 路、个位 2 路 |

大小规则：

- 小：`0-4`
- 大：`5-9`

012 路规则：

- 0 路：除以 3 余 0，例如 `0、3、6、9`
- 1 路：除以 3 余 1，例如 `1、4、7`
- 2 路：除以 3 余 2，例如 `2、5、8`

### 组选统计读法：默认关闭

组选统计读法默认关闭，必须点击“开启”后才显示按钮并参与筛选。

它不看百十个位顺序，只统计数量，所以 `123` 和 `321` 会被当作同一类统计结果。

支持：

| 条件 | 示例 | 含义 |
| --- | --- | --- |
| 大小比 | 2大1小 | 三位中有 2 个大号、1 个小号 |
| 奇偶比 | 2奇1偶 | 三位中有 2 个奇数、1 个偶数 |
| 质合比 | 2质1合 | 三位中有 2 个质数、1 个合数 |
| 012 路比 | 0路1 1路1 2路1 | 三位数字分别命中 0 路、1 路、2 路各 1 个 |

关闭状态下，即使之前选过组选统计条件，也不会提交到筛选请求。加载历史记录时，如果历史本身包含组选统计条件，系统会自动开启，避免历史条件丢失。

## 7. 统计特征

统计特征已经拆成三块，便于阅读和操作。

### 和值筛选

- 和值：三位数字之和，范围 `0-27`。
- 和尾：和值个位，范围 `0-9`。
- 平均值：三位数字平均值的近似整数，范围 `0-9`。

### 跨度筛选

- 跨度：最大数字减最小数字，范围 `0-9`。
- 首尾距：百位与个位差值，范围 `0-9`。
- AC 值：号码内部差值离散度。

### 两码筛选

- 二码和
- 二码和尾
- 二码差
- 两码合差
- 二码组合

两码类规则会同时考虑三位号码中的任意两位组合。

## 8. 区间组合

### 段组条件

支持同时添加多个段组规则：

| 段组 | 分组结构 | 通过逻辑 |
| --- | --- | --- |
| 2-2-6 | 2 个、2 个、6 个 | 去重数字最多落入 2 段 |
| 2-3-5 | 2 个、3 个、5 个 | 去重数字最多落入 2 段 |
| 3-3-4 | 3 个、3 个、4 个 | 去重数字最多落入 2 段 |
| 5-5 | 5 个、5 个 | 去重数字必须跨两段 |

每条段组规则都必须完整覆盖 `0-9`，并且同一条规则内数字不能重复。

### 段组容错

段组容错默认是“严格”。

- 严格：所有段组规则都必须通过。
- 容错 1：允许 1 条段组规则不满足。
- 容错 2：允许 2 条段组规则不满足。

容错只作用于段组规则，不影响 N 码、胆码、杀码、形态等其他条件。

### N 码命中

N 码用于选择一组数字，然后判断号码命中这组数字的数量。

支持码数：`3-8`。

支持条件：

| 条件 | 含义 |
| --- | --- |
| =01 | 命中 0 个或 1 个 |
| =012 | 命中 0 个、1 个或 2 个 |
| =123 | 命中 1 个、2 个或 3 个 |
| =23 | 命中 2 个或 3 个 |

同号计重设置：

- 默认：同号不计重。例：`112` 命中数字 `1` 时，只算 1 个不同数字。
- 可选：同号计重。例：`112` 命中数字 `1` 时，算 2 次。

## 9. 结果区

筛选完成后，右侧会展示：

- 号码池总数
- 筛选命中数
- 命中占比
- 已启用条件数
- 号码墙
- 筛选步骤
- 历史记录

结果区按钮：

- 复制号码：复制当前号码墙结果。
- 保存历史：保存当前筛选结果，后续可回查。

结果排版支持：

- 每行数量
- 分隔符：双空格、空格、逗号、顿号、逐行

如果修改了筛选条件但没有重新筛选，结果区会显示“条件已变更，请重新筛选”的提示。

## 10. 历史与开奖回查

历史记录保存在服务端文件：

```text
history_records.json
```

开奖数据保存在：

```text
draw_records.json
```

点击更新开奖后，系统会拉取最新开奖数据并合并到本地。历史回查时：

- 直选大底按顺序精确匹配。
- 组选大底、组三、组六等按组选逻辑匹配，不看顺序。

历史记录可以修改期号，便于回查指定开奖期。

## 11. 接口说明

### GET `/api/draws`

读取本地开奖数据，返回最近开奖记录、最新期号、下一期号和本地记录数量。

### POST `/api/update_draws`

拉取外部开奖数据，合并写入 `draw_records.json`。

### POST `/api/filter`

核心筛选接口。

示例：

```json
{
  "base_mode": "direct",
  "text": "",
  "digits": "357",
  "kill_digits": "48",
  "segment_tolerance": 1,
  "segment_filters": [
    {
      "mode": "2-3-5",
      "enabled": true,
      "groups": ["01", "234", "56789"]
    }
  ],
  "code_filters": [
    {
      "code_len": 5,
      "condition": "012",
      "digits": "01234",
      "count_repeat": false,
      "enabled": true
    }
  ],
  "rule_filters": [
    {
      "type": "odd_even_count",
      "mode": "include",
      "values": ["2:1"],
      "enabled": true
    }
  ],
  "position_filter": {
    "include": ["", "", ""],
    "exclude": ["", "", ""]
  },
  "shape_filter": {
    "mode": "include",
    "types": [],
    "prime_count": [],
    "values": []
  }
}
```

常用 `rule_filters.type`：

| 类型 | 含义 |
| --- | --- |
| `sum` | 和值 |
| `sum_tail` | 和尾 |
| `average` | 平均值 |
| `span` | 跨度 |
| `first_last_diff` | 首尾距 |
| `ac` | AC 值 |
| `pair_sum` | 二码和 |
| `pair_sum_tail` | 二码和尾 |
| `pair_diff` | 二码差 |
| `pair_sum_diff` | 两码合差 |
| `pair_code` | 二码组合 |
| `mod012` | 012 路直选读法 |
| `big_small` | 大小直选读法 |
| `odd_even` | 奇偶直选读法 |
| `prime_composite` | 质合直选读法 |
| `big_small_count` | 大小比组选统计读法 |
| `odd_even_count` | 奇偶比组选统计读法 |
| `prime_composite_count` | 质合比组选统计读法 |
| `mod012_count` | 012 路比组选统计读法 |

返回内容包含：

- `total`：号码池总数
- `filtered`：筛选后的号码列表
- `count`：命中数量
- `percent`：命中占比
- `steps`：筛选步骤
- `segment_tolerance`：段组容错数
- `rule_filters`、`segment_filters`、`code_filters` 等回显条件

### POST `/api/generate_codes`

复式码生成接口。

```json
{
  "codes": [
    {"digits": "01234", "len": 5},
    {"digits": "123456", "len": 6}
  ]
}
```

返回生成后的号码列表、数量和规范化后的复式码信息。

### GET `/api/history`

读取历史筛选记录。

### POST `/api/history`

保存历史筛选记录。

### DELETE `/api/history`

清空历史筛选记录。

### POST `/api/check_history`

根据本地开奖数据回查历史记录是否命中。

```json
{
  "items": [
    {
      "targetIssue": "2026126",
      "base_mode": "direct",
      "filtered": ["123", "456"]
    }
  ]
}
```

## 12. 数据文件

| 文件 | 作用 |
| --- | --- |
| `draw_records.json` | 本地开奖数据 |
| `history_records.json` | 筛选历史记录 |
| `app.py` | 后端接口和筛选逻辑 |
| `index.html` | 前端界面和交互逻辑 |
| `tests/` | 自动化测试 |

## 13. 注意事项

- 不同筛选条件之间是 AND 关系，条件越多，结果越少。
- 同一个数字选择器内部，多选表示命中任意一个选项。
- 组选统计读法默认关闭，需要手动开启才参与筛选。
- 直选读法会按百位、十位、个位读取，不等同于组选统计。
- 段组容错只对段组条件生效。
- 开奖数据依赖外部接口，网络异常时可能更新失败，可保留本地旧数据继续筛选。
