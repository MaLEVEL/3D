# 福彩3D组选筛号工具说明

这是一个零依赖本地 Web 工具，用于从号码池中筛选福彩3D组选号码，并保存历史记录后按期开奖号更新中奖状态。

## 运行方式

在当前目录执行：

```powershell
python app.py
```

服务启动后访问：

```text
http://127.0.0.1:5000
```

## 筛选条件

### 号码池

在“号码池”输入框粘贴任意文本，系统会自动提取所有三位数字，例如：

```text
012 013 014 001 112
```

重复号码会自动去重，并保留第一次出现的顺序。

### 筛选数字

填写如 `357`，表示号码中只要包含 `3`、`5`、`7` 任意一个数字就保留。

### 两码差数字

填写如 `523`，系统会计算三位号码中：

```text
|百位-十位|、|百位-个位|、|十位-个位|
```

只要任意一个差值命中填写数字，就保留。

## 段组筛选

当前支持三种段组：

```text
2-2-6
2-3-5
5-5
```

页面上三种段组会同时显示，每一种都有“启用”开关。可以只启用一种，也可以同时启用多种。

启用多个段组时，筛选结果必须同时通过所有启用段组。

段组必须满足：

- 每个数字 `0-9` 在同一个段组类型内只能属于一个分组。
- 每个启用的段组类型都必须覆盖 `0-9`。
- 每组数字数量必须和段组类型一致。

规则解释：

- `2-2-6`、`2-3-5`：组选号码不能同时落满三段。也就是一个号码的去重数字最多只能覆盖两个段。
- `5-5`：组选号码必须跨两段。也就是不能全部落在同一个五码组内。

示例：

```text
2-3-5 段组：01 | 234 | 56789
```

如果号码是 `027`，去重数字覆盖 `01`、`234`、`56789` 三段，会被过滤。

如果号码是 `055`，去重数字只覆盖 `01` 和 `56789` 两段，会保留。

## 期号与历史记录

筛选区不需要手动填写期号。

点击“保存历史”时，系统会根据当天日期自动生成期号名，格式为：

```text
YYYYMMDD
```

例如 2026年5月13日 保存的记录，默认期号名为：

```text
20260513
```

如果需要改成真实开奖期号，可以在“筛选历史”里直接修改该条记录的期号，然后点击“改期号”。

## 更新中奖记录

点击“更新中奖记录”后，系统会：

1. 调用开奖接口获取最新一期福彩3D开奖号。
2. 保存到本地 `draw_records.json`。
3. 遍历历史记录。
4. 如果历史记录的期号和本地开奖数据期号一致，就回填开奖号。
5. 按组选逻辑判断是否中奖。

组选命中判断会忽略顺序，例如：

```text
保存结果包含 357
开奖号为 753
```

会判定为中奖。

## 本地数据文件

工具会在当前目录维护：

```text
draw_records.json
```

这个文件保存已获取到的开奖数据，最多保留最近 500 条。

历史筛选记录保存在浏览器 localStorage 中，不会写入服务器文件。

## 接口说明

### POST `/api/filter`

用于筛选号码。支持同时传多个段组条件：

```json
{
  "text": "012 013 014",
  "digits": "357",
  "advance_digits": "23",
  "segment_filters": [
    {
      "mode": "2-3-5",
      "enabled": true,
      "groups": ["01", "234", "56789"]
    },
    {
      "mode": "5-5",
      "enabled": true,
      "groups": ["01234", "56789"]
    }
  ]
}
```

### POST `/api/update_draws`

获取最新开奖数据并保存到本地。

### POST `/api/check_history`

按本地开奖数据回查历史记录中奖状态。

```json
{
  "items": [
    {
      "targetIssue": "2026123",
      "filtered": ["357", "358"]
    }
  ]
}
```

## 注意事项

- 开奖更新依赖外部接口，网络不可用或接口异常时会提示失败。
- 当前工具只判断组选是否命中，不区分直选顺序。
- 浏览器清理站点数据后，筛选历史会被清空。
