# 福彩3D组选筛号工具

零依赖本地 Web 工具，用于从号码池中筛选福彩3D组选号码，支持多段组条件组合筛选，保存历史记录后可按开奖号回查中奖状态。

## 运行方式

```powershell
python app.py
```

浏览器自动打开 `http://127.0.0.1:5000`。

## 页面布局

左侧侧边栏 + 右侧主内容区（窄屏自动上下排列）：

- **侧边栏**：开奖概览（最新期号、下一期号、开奖号、本地条数）+ 近10期开奖列表 + “更新开奖”按钮
- **主内容区**：筛选配置 → 统计面板 → 筛选结果 → 筛选历史

## 筛选条件

所有筛选条件可任意组合，多个条件之间为 **AND** 关系（号码须同时满足全部条件）。

### 号码池

粘贴任意文本，自动提取所有三位数字（`000`-`999`），去重并保留出现顺序。

### 胆码筛选

如填 `357`，号码中包含 `3`、`5`、`7` 任意一个即保留。

### 杀码筛选

如填 `48`，号码中包含 `4` 或 `8` 任意一个即排除。杀码优先级最高，先于所有其他条件执行。

### 两码差数字

| 百-十 |、| 百-个 |、| 十-个 |，任一差值的绝对值命中即保留。

如填 `5`，差值计算结果中有 `5` 的号码保留。

### 复式码生成

选择码数（5/6/7码）并输入对应数量的不同数字，自动生成所有组六 + 组三组合。可添加多组不同码数，合并去重后填入号码池。

| 码数 | 组六       | 组三    | 总注数 |
|------|------------|---------|--------|
| 5码  | C(5,3)=10  | 5×4=20  | 30     |
| 6码  | C(6,3)=20  | 6×5=30  | 50     |
| 7码  | C(7,3)=35  | 7×6=42  | 77     |

### 段组筛选

支持三种段组类型，可同时添加多个（筛选结果须通过所有段组）：

| 类型 | 分组 | 规则 |
| ---- | ---- | ---- |
| 2-2-6 | 2码+2码+6码 | 去重数字不能落满三段 |
| 2-3-5 | 2码+3码+5码 | 去重数字不能落满三段 |
| 5-5 | 5码+5码 | 去重数字必须跨两段 |

约束：每个段组必须完整覆盖 `0-9`，分组间数字不可重叠，每组数量不可超出预设值。

示例——2-3-5 段组 `01 | 234 | 56789`：

- `027` → 覆盖三段 → **过滤**
- `055` → 只覆盖 `01` 和 `56789` 两段 → **保留**

### N码条件

选择 N 个数字（N 范围 3~8）和一个命中范围，号码中属于这 N 个数字的**不同数字个数**在范围内即通过。

| 条件 | 含义 |
| ---- | ---- |
| =01 | 号码中恰好包含 0 个或 1 个所选数字 |
| =012 | 号码中恰好包含 0~2 个所选数字 |
| =123 | 号码中恰好包含 1~3 个所选数字 |

示例——5码=012，数字 `14578`：

- `114` → 不同数字 {1,4} 命中2个 → **保留**（注：1 出现两次只算一个不同数字）
- `026` → 不同数字 {0,2,6} 命中0个 → **保留**
- `145` → 不同数字 {1,4,5} 命中3个 → **过滤**

可添加多组N码条件，筛选结果须同时通过所有N码条件。

## 期号与历史记录

保存历史时优先使用开奖记录推算的下一期号（如 `2026124`）；若无本地开奖数据则回退为当日日期 `YYYYMMDD`。

历史记录中可随时修改期号——点击记录中的”改期号”即可。

## 更新开奖 / 更新中奖记录

侧边栏”更新开奖”和历史区”更新中奖记录”都会：

1. 调用外部接口获取最新开奖数据，合并到本地 `draw_records.json`
2. 刷新侧边栏概览和近10期列表
3. 遍历历史记录，回填开奖号并按组选逻辑判定中奖

## 本地数据

- `draw_records.json`：开奖数据（最多 500 条）
- `history_records.json`：筛选历史（最多 80 条），存储在服务器端

## 接口说明

### GET `/api/draws`

读取本地开奖记录，返回最近 10 期及概览信息。

### POST `/api/filter`

```json
{
  "text": "012 013 014",
  "digits": "357",
  "kill_digits": "48",
  "advance_digits": "23",
  "segment_filters": [
    {"mode": "2-3-5", "enabled": true, "groups": ["01", "234", "56789"]},
    {"mode": "5-5", "enabled": true, "groups": ["01234", "56789"]}
  ],
  "code_filters": [
    {"code_len": 5, "condition": "012", "digits": "14578", "enabled": true}
  ]
}
```

返回：

```json
{
  "total": 30,
  "filtered": ["015", "016", "..."],
  "count": 24,
  "percent": 80.0,
  "advance_digits": "23",
  "segment": "2-3-5 segment [01 | 234 | 56789]; 5-5 segment [01234 | 56789]",
  "segment_filters": [
    {"mode": "2-3-5", "groups": ["01", "234", "56789"]},
    {"mode": "5-5", "groups": ["01234", "56789"]}
  ],
  "code_desc": "5码=012[14578]",
  "code_filters": [
    {"code_len": 5, "condition": "012", "digits": "14578"}
  ]
}
```

### POST `/api/generate_codes`

```json
{
  "codes": [
    {"digits": "01234", "len": 5},
    {"digits": "56789", "len": 5}
  ]
}
```

返回：

```json
{
  "ok": true,
  "generated": ["001", "002", "..."],
  "count": 60,
  "codes": [{"digits": "01234", "len": 5}, {"digits": "56789", "len": 5}]
}
```

### POST `/api/update_draws`

从外部抓取最新开奖数据并存入本地。

### POST `/api/check_history`

```json
{
  “items”: [
    {“targetIssue”: “2026123”, “filtered”: [“357”, “358”]}
  ]
}
```

按本地开奖数据回填中奖状态（组选匹配，忽略顺序）。

## 注意事项

- 开奖数据需要手动点击”更新开奖”获取，或直接编辑 `draw_records.json`
- 外部开奖接口可能因网络或限流不可用，此时可手动填入
- 筛选历史存储在服务器端 `history_records.json`，不受浏览器清除数据影响
